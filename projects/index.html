Network Science Research Tool


[ASU, Fall 2016]


Summary

Today, we all know the importance of connectivity or being online. If we weren't connected now, you were not be reading this. Millions of emails, messages, confidential information are travelling via network from one node to another on daily basis. Even if this network goes down for few seconds, world comes to a stand still. And there are many kind of disasters happeing around like Tsunami, Hurricane, Floods etc., may affect the connectivity of networks. Thus, it is important to have a robust network which is immune to such faults. But the big challenge here is, as network might be over a vast geographical area, it is difficult to test the robustness of the network. In this project, we have build a tool (based on a approach given in a research paper published by Dr. Sen and team) that helps us estimate the impact of such disasters(fault) on a given network.

My Contribution

  • Designed the high level architecture of the web application
  • Intergarated the Google Maps in the front end and made it fully interactive so that user can design a new network, edit existing one, delete/update nodes, edges in the graph etc.
  • Desgined the database for the application
  • Developed different request handlers(server as well as client side) to and from server
  • Used results from different algorithms(developed by other contributors) for testing generic and specified fault for a given network infrastructure
  • Integration of differet modules, in and out testing and deployement of the web app on ASU server

Extraction, Ranking & Indexing of Multimedia Features


[ASU, Fall 2016]


Summary

We are living in an era where information/data is increasing substantially on daily basis and we need to organize, store that information so that it can be used later by the users. And when we say word “Information”, it can be of any type like numerical, text based, image, video etc., also it can be further divided in to two categories i.e. relevant and irrelevant information. So here we are dealing with two situations i.e. first, how to go about organize and storing this information so that it can be retrieved efficiently later on and second, how to get the perfect and optimized result from the stored information. This project is mainly focused on second part where we have used different feature extraction techniques which we are going to discuss further in detail in different sections. And using these extraction techniques we will try to compute the distance and similarity between the two multimedia objects. And later on using PageRank Algorithm, we will create a relevancy graph and rank them. Also, we have used some dimension reduction and indexing techniqes to make search more efficient

My Contribution

  • Extracted different multimedia(video) features like histogram, sift vectors, motion vectors using MATLAB
  • Performed subsequence search for similar frames on a multimedia database based on different similarity measures like Cosine Similarity, Ecludian distance, Mahalanobis distance etc.
  • Generated similarity graph for video frames and found most significant frames using PageRank Algorithm
  • Performed dimensionality reduction using PCA & LSH for fast query processing
