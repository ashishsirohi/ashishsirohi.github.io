ASU Class Notifier


[Summer 2017]


Summary

Arizona State University is huge, home to approximately 80K students from around the world. And it offers a large variety of courses across different domains to accommodate all the students but sometimes, few courses are red hot, maybe because of the professor or maybe it's their last chance to get the course as they are graduating or it's a degree requirement course etc. but it's not possible to accommodate all. So the only hope of getting the course is if somebody changes his/her mind and drops it. But ASU doesn't have any system which can notify students if a course has been dropped or something similar.

In order to grab that open course, you have to be on the course site all the time and check it continuously. Lots of waste of time, right? and even after that, there is no guarantee that will get that course. I myself faced the issue and that's how I came up with the idea of writing a script which will track the status of the specified course and notify me once it is available. I wrote it, it worked and I got the course. But then I thought, why not help other students as everyone is not from Computer Science background or write code. So I turned my script into a fully functional web application where users can sign up/login in and register for the courses they want to get notified for.

App features/My Work

  • User login/Signup implemented using Django Authentication system
  • User can search for the course availability, if not available, they can add that course to their notification list
  • User can manage their notification list by adding more courses or deleting existing ones
  • A Python script, running periodically using heroku scheduler to check the status of the specified course, shares the same Postgres database as of the Django Web App
  • Multithreading is being used to check the status of multiple courses in parallel if course is available, an email is being sent to all the users who have this course on their notification list
  • Designed the database efficiently to avoid unnecessary use of resources
  • Used PhantomJS web driver(optimized it) along with BeautifulSoup for efficient web page scrapping

Tech Used:

Python, Django with Authentication Systems & Django Forms, PostgreSQL, JavaScript, HTML, Bootstrap, jQuery, PhantomJS, Selenium Package, BeautifulSoup, heroku

Network Science Research Tool


[ASU, Fall 2016]


Summary

Today, we all know the importance of connectivity or being online. If we weren't connected now, you were not be reading this. Millions of emails, messages, confidential information are traveling via the network from one node to another on daily basis. Even if this network goes down for few seconds, the world comes to a standstill. And there are many kinds of disasters happening around like Tsunami, Hurricane, Floods etc., may affect the connectivity of networks. Thus, it is important to have a robust network which is immune to such faults. But the big challenge here is, as the network might be over a vast geographical area, it is difficult to test the robustness of the network. In this project, we have build a Web-based tool (based on an approach given in a research paper published by Dr. Sen and team) that helps us estimate the impact of such disasters(fault) on a given network.

My Contribution

  • Designed the high level architecture of the web application
  • Intergarated the Google Maps in the front end and made it fully interactive so that user can design a new network, edit existing one, delete/update nodes, edges in the graph etc.
  • Desgined the database for the application
  • Developed different request handlers(server as well as client side) to and from server
  • Used results from different algorithms(developed by other contributors) for testing generic and specified fault for a given network infrastructure
  • Integration of differet modules, in and out testing and deployement of the web app on ASU server

Extraction, Ranking & Indexing of Multimedia Features


[ASU, Fall 2016]


Summary

We are living in an era where information/data is increasing substantially on daily basis and we need to organize, store that information so that it can be used later by the users. And when we say word “Information”, it can be of any type like numerical, text based, image, video etc., also it can be further divided into two categories i.e. relevant and irrelevant information. So here we are dealing with two situations i.e. first, how to go about organize and storing this information so that it can be retrieved efficiently later on and second, how to get the perfect and optimized result from the stored information. This project is mainly focused on second part where we have used different feature extraction techniques which we are going to discuss further in detail in different sections. And using these extraction techniques we will try to compute the distance and similarity between the two multimedia objects. And later on using PageRank Algorithm, we will create a relevancy graph and rank them. Also, we have used some dimension reduction and indexing techniques to make search more efficient

My Contribution

  • Extracted different multimedia(video) features like histogram, sift vectors, motion vectors using MATLAB
  • Performed subsequence search for similar frames on a multimedia database based on different similarity measures like Cosine Similarity, Euclidian distance, Mahalanobis distance etc.
  • Generated similarity graph for video frames and found most significant frames using PageRank Algorithm
  • Performed dimensionality reduction using PCA & LSH for fast query processing

Tech Used:

Python, MATLAB Scripting, Visual basics

IP Reflectore


[ASU, Feb 2017]


Summary

The goal of this project was to create a “reflector” using python-scapy package which will relaunch attacks sent to a given IP address and ethernet address to the IP address that sent the attack. This acted as a mirror, such that when an adversary is portscanning a network, they will actually be port scanning themselves. When they launch an exploit at the reflector, the attack will be reflected back at them.

My Contribution

  • Developed an IP reflector which detects all kinds of data packets sent to victim and reflects them back to the attacker
  • Exploited Python-Scapy package for creating spoofed packets to fool the attacker

HTTP 1.1 Server in C with Backdoor functionality


[ASU, Feb 2017]


Summary

A critical part of establishing persistence on a system is to leave a “backdoor” that allows the hacker access to the system at a later date, without exploiting the same vulnerabilities (as they may be fixed in the meantime). In this mini project, I have written a backdoor that pretends to be a web server. The goal of this mini project was to build an HTTP 1.1 server from scratch in C based on RFC 2616 with backdoor functionality. The backdoor functionality is that when your server receives a GET request for a URL in the form of /exec/<command>, then your server should take <command> and execute it using the system Linux system call and the HTTP response will be the stdout of the executed command.

Analyzing Geospatial Data and Finding NYC Taxi Hotspots using GeoSpark


[ASU, Spring 2017]


Summary

We are living in an era where millions of people are going online each day and a huge amount of data is being generated and a major part of the data is location based. It would be interesting to perform certain analysis and operations on the top of it to make some sense out of this huge data as it can benefit us in solving some major social issues as well as it may benefit private organizations. In this project, we have mainly focused on the spatial data to perform certain geospatial operations on the top of it. It is very important to choose an efficient system for doing such operations on a huge data set as geospatial operations are expensive, also we need to make sure that our system is scalable, can perform in-memory computation efficiently, can leverage the benefits of distributed systems. GeoSpark being an extension to the Spark for doing efficient in-memory operations on the spatial data will be an interesting choice for doing all spatial operations efficiently. We were also motivated to see how different algorithms can affect the system performance in the different type of spatial operations.

My Contribution

  • Configured HDFS with one master and two slaves systems
  • Created GeoSpark SpatialRDD (PointRDD) from a given data set and performed Spatial operation like Range Query, KNN Query, Join Query etc on the top of it
  • Written Spatial join query using cartesian product and compared the result with in-built geospark spatial join query(using equijoin)
  • Implemented a Spark program to calculate the Getis-Ord statistic of NYC Taxi Trip datasets

Tech Used:

Java, Hadoop, Spark, GeoSpark, Ganglia
